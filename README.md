## 项目简介

`Backpack_Design` 是一个基于 **Cocos Creator 3.8.7** 开发的背包系统交互原型，主要使用 **TypeScript** 编写。项目围绕游戏背包常见需求，实现了物品配置、背包分页、拖拽交换、物品堆叠、筛选搜索、合成、使用、丢弃、场景拾取、本地存档等功能。

项目中使用到的工具和 AI：

- 游戏引擎：Cocos Creator 3.8.7
- 开发语言：TypeScript
- 资源处理：PowerShell 脚本
- 生图 AI：即梦
- 音效 AI：meowa.ai
- 主要使用的 Cocos 能力：Prefab、Inspector 配置、UI 组件、Tween 动画、AudioSource 音效、localStorage 存档

## 项目做了什么

- 实现了 4x4 背包格子界面，支持多页背包、翻页和页码圆点跳转。
- 实现了物品显示，包括图标、数量、稀有度边框、选中态和空格状态。
- 实现了物品交互，包括点击选择、拖拽交换、跨页拖放、自动堆叠、整理和合成。
- 实现了筛选搜索，可以按物品分类、稀有度和名称过滤背包内容。
- 实现了右键菜单，可以选择使用或丢弃物品，并通过数量弹窗控制操作数量。
- 实现了场景物品测试区，场景物品可以拾取进背包，背包物品也可以丢弃回场景。
- 实现了本地存档，保存当前页、物品 ID、物品数量和背包格子状态。
- 实现了 UI 音效和提示框，提升背包打开、关闭、翻页、交换、整理等操作反馈。
- 实现了资源处理脚本，用于批量处理背包 UI 和物品图标资源。

## 怎么做的

项目核心代码位于 `assets/scripts/inventory`：

- `BackpackController.ts`：背包主控制器，负责分页、渲染、拖拽、筛选、合成、使用、丢弃、存档和运行时 UI。
- `ItemSlot.ts`：单个背包格组件，负责显示物品图标、数量和稀有度框。
- `ItemCatalog.ts`：物品配置系统，负责管理武器、防具、消耗品、材料、任务物品和其他物品。

运行时会先读取 Cocos Inspector 中配置的物品数据，建立物品 ID 索引，然后读取本地存档。如果没有存档，就根据配置生成演示背包。每次玩家进行拖拽、交换、筛选、合成、使用、丢弃或扩容后，背包都会重新渲染并保存状态。

## 重点实现

### 1. 数据驱动的物品配置

物品不写死在代码数组里，而是通过 Cocos Inspector 配置。物品 ID 可以不连续，但必须唯一。背包存档只保存 `itemId + count`，名称、图标、稀有度、描述、效果和合成配方都从配置表读取。

### 2. 完整拖拽手感

拖拽系统支持触发距离、指针偏移、拾取缩放、透明度、阻尼跟随、速度倾斜、目标格高亮、落格动画和跨页拖放。拖拽参数集中在 `DragFeelInspectorConfig` 中，方便调试手感。

### 3. 筛选视图和真实数据分离

筛选不会直接改动真实背包数组，而是生成一层可视索引。这样分类、稀有度、搜索和翻页可以组合使用，同时交换、使用、丢弃和合成仍然能正确作用到真实库存。

### 4. 使用效果预留接口

物品使用时会派发事件：

- `inventory-item-used`
- `inventory-item-effect`

后续可以让角色属性、战斗、任务、装备或成就系统监听这些事件，实现真正的游戏效果。

### 5. 场景物品闭环

项目加入了场景物品测试区，用来验证“场景物品 -> 背包 -> 丢弃回场景”的完整流程。拾取时会优先补齐已有堆叠，再放入空格；丢弃物会在测试区停留一段时间后消失。

## 做过的优化

- 使用 `Map` 建立物品 ID 索引，避免数组下标绑定物品 ID。
- 启动时校验物品配置，提示重复 ID、非法 ID、缺少名称、缺少图标等问题。
- 存档带版本号，读取失败时自动回退到演示数据。
- 读取存档时会根据当前最大堆叠数自动修正数量。
- 操作按钮会根据当前状态自动启用或禁用，减少无效操作。
- 筛选栏、稀有度菜单、页码圆点、提示框、右键菜单和数量弹窗由代码运行时生成，减少场景中的重复节点。
- 拖拽、翻页、分类切换和页码选中都加入 Tween 动画反馈。
- 资源处理脚本统一输出尺寸和透明背景，减少手动裁切和对齐成本。

## 如何运行

1. 使用 Cocos Creator 3.8.7 打开项目。
2. 打开场景 `assets/scenes/Backpack-scene.scene`。
3. 点击预览运行。
4. 在预览窗口中测试背包打开、拖拽、筛选、合成、使用和丢弃功能。

## 还差什么

- 还没有完成系统化压测，需要模拟大量物品、大容量背包、高频拖拽、频繁筛选和连续存档等场景。
- 还没有补充完整测试用例，例如物品配置校验、合成扣料、堆叠上限、筛选结果、跨页拖拽、存档读取和异常数据恢复。
- 为了应对后续压测，还需要继续优化算法和数据结构，例如减少频繁全量遍历、优化筛选索引、缓存分类结果、降低频繁渲染和存档写入成本。
- 物品效果目前只派发事件，还没有接入真实角色属性、战斗、装备栏或任务系统。
- 背包扩容目前是测试功能，没有消耗金币、材料或道具。
- 场景掉落物只用于测试，还没有接入真实地图坐标、碰撞、拾取范围或网络同步。
- 存档目前使用 `localStorage`，正式项目还需要账号存档、云存档或本地文件存档方案。
- UI 当前主要适配 1280x720 演示场景，还需要继续做多分辨率和移动端适配。
